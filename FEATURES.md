# MatchSnapper — Функциональность (текущий этап)

## Авторизация

- **Регистрация** — email, имя пользователя, пароль (мин. 8 символов)
- **Вход** — по email + пароль, выдаёт access + refresh токены
- **Автообновление токена** — axios-интерцептор перехватывает 401 и прозрачно обновляет токен через refresh; параллельные запросы ставятся в очередь
- **Выход** — очищает токены из localStorage и store

---

## Профиль пользователя

### Собственный профиль (`/profile`)
- Просмотр: аватар, имя пользователя, email
- Редактирование **bio** (короткое «о себе»)
- **Загрузка аватара** — клик по аватару открывает выбор файла
  - Допустимые форматы: JPEG, PNG, WebP, GIF
  - Максимальный размер: 5 МБ
  - Бэкенд автоматически кропает и ресайзит до 256×256 px JPEG
  - При загрузке нового файла кеш браузера сбрасывается через `?t=timestamp`
  - Валидация формата и размера происходит на фронте до отправки с понятными сообщениями об ошибке
- **Удаление аватара** — кнопка × в правом верхнем углу аватара

### Профиль другого пользователя (`/users/:id`)
- Аватар, имя, bio
- **Блок совместимости**: процент совпадения + прогресс-бар
  - Иконка `?` с поповером при наведении — объясняет алгоритм расчёта
  - Список общих интересов с весами: «Мой вес / вес пользователя»
- **Слепок пользователя** — полный список его тегов с весами
- Кнопка **«Написать»** — отправляет запрос на контакт (заглушка, MVP)

## Слепок (`/snapshot`)

Слепок — набор тегов с весами, описывающий интересы пользователя.

**Веса:** S (5) → A (4) → B (3) → C (2) → D (1)

- Просмотр текущего слепка
- **Добавление тега** — поиск с автодополнением по каталогу; если тег не найден — можно создать новый прямо из поиска
- **Изменение веса** — пикер S/A/B/C/D прямо в строке, сохраняется мгновенно
- **Удаление тега** — кнопка × в строке
- **Порядок тегов** — сохраняется в том порядке, в котором пользователь их добавил; порядок персистируется в `localStorage` и восстанавливается после перезагрузки
- **Перетаскивание** — drag-and-drop переупорядочивание строк; новый порядок сохраняется сразу
- **Кнопка «По приоритету»** — переключает отображение в порядке убывания веса (S→D); не меняет сохранённый пользовательский порядок; drag-and-drop при активной сортировке отключён

## Матчинг (`/matches`)

- Список пользователей, отсортированных по схожести слепков
- **Непросмотренные — выше** просмотренных при одинаковом score
- Пагинация (20 результатов на страницу)
- Каждая карточка показывает:
  - Аватар, имя, bio
  - Процент совместимости + прогресс-бар
  - Общие теги с весами обоих пользователей
- **Клик по карточке** → переход на профиль пользователя
- **«Отметить просмотренным»** — помечает профиль как просмотренный (кнопка в карточке, не вызывает переход)
- **«Написать»** — отправляет запрос на контакт (заглушка, MVP; кнопка в карточке)

---

## Алгоритм совместимости

Косинусное сходство векторов слепков: каждый тег — отдельная ось, вес — значение по этой оси. Результат — число от 0 до 1, отображается как процент.

```
User A: { футбол:S=5, музыка:B=3 }  → вектор [5, 3, 0]
User B: { футбол:A=4, чтение:C=2 }  → вектор [4, 0, 2]

score = dot(A,B) / (|A| × |B|) = 20 / (√34 × √20) ≈ 0.77 → 77%
```

## Каталог тегов

- ~51 предустановленный тег, 8–10 категорий (спорт, культура, технологии и др.)
- Поиск по названию и категории
- Любой авторизованный пользователь может добавить новый тег

## Навигация

| Маршрут | Страница |
|---|---|
| `/login` | Вход |
| `/register` | Регистрация |
| `/matches` | Список матчей (главная) |
| `/snapshot` | Редактор слепка |
| `/profile` | Свой профиль |
| `/users/:id` | Профиль другого пользователя |

Неавторизованный пользователь редиректится на `/login`. Авторизованный — с `/login` и `/register` на `/matches`.

## Что запланировано, но не реализовано (MVP-заглушки)

- **Чат / контакт** — таблица `contact_requests` создана, эндпоинт работает, но UI только показывает статус «Запрос отправлен». Полноценный чат — следующий этап.
- **Модерация тегов** — любой пользователь может создать тег без проверки.
- **OAuth** — структура `AuthProvider` ABC готова к расширению (Google, VK и др.), но реализован только JWT.
- **pgvector** — матчинг считается в Python/numpy; при масштабировании можно перейти на pgvector без изменения API.
